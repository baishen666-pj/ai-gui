> **版本:** 2.0 (2.5D SVG 版)

> **承接团队:** Hermes AI 游戏团队
> **风格方向:** 等轴测 2.5D + 动漫 Q 版

## 1. 当前状态

项目已从 Three.js/R3F 迁移到纯 SVG + CSS 的 2.5D 等轴测视图。当前实现使用简单几何 SVG 元素构建场景，视觉效果功能正确但缺乏专业感。

### 当前文件结构

```
src/renderer/src/components/three/
├── IsoEngine.ts          (46行) — 等轴测投影数学 + 深度排序
├── IsoScene.tsx          (304行) — 主场景：动画循环、审批走向、缩放/平移
├── IsoRoom.tsx           (89行)  — 菱形地板 + 墙壁 + 网格 + 窗户

├── IsoFurniture.tsx      (105行) — 7 种简单 SVG 家具
├── IsoCharacter.tsx      (330行) — Q 版角色 + 5 种动画 + 表情系统
├── IsoEffects.tsx        (74行)  — 审批光环 + 行走拖尾
```

### 当前视觉效果

- 角色：纯几何 SVG（圆头 + 方身 + 方腿），可区分 6 种 role
- 家具：简单矩形/椭圆组合（方盒子感强）
- 颜色：主题感知但缺乏层次和质感
- 动画：SVG 属性驱动（呼吸、打字、行走摆臂等）

---

## 2. 设计目标

### 核心原则

1. **一眼识别** — 6 种角色类型无需看标签即可区分
2. **专业质感** — 告别方盒子，增加细节层次、光影、纹理
3. **性能优先** — SVG 方案必须保持 52KB 级别，60fps 无卡顿
4. **三主题适配** — dark / light / cyberpunk 三套色板下都好看

### 风格参考

- 等轴测像素风（如 Opus Magnum、Factorio 的建筑风格）
- 日系 Q 版办公漫画（如《工作中的恶魔》）
- Dribbble 上的 isometric office 插画
- 低多边形但细节丰富的矢量插画风格

## 3. 角色升级规格

### 当前角色结构（IsoCharacter.tsx）

```svg
<!-- 现状：纯几何拼接 -->
<circle cx="0" cy="22" r="14" />  <!-- 头：简单圆 -->
<rect x="-10" y="32" width="20" height="26" rx="4" />  <!-- 身：方盒 -->
<rect x="-2.5" y="0" width="5" height="16" rx="2" />   <!-- 腿：方条 -->
```

### 目标：动漫 Q 版 SVG 角色

#### 身体比例（Q 版 1:2.5 头身比）

| 部位 | 尺寸 | 说明 |
|------|------|------|
| 头部 | 16x16 单位 | 占全身 40%，大眼小嘴 |
| 身体 | 14x18 单位 | 缩短躯干，强化 Q 版感 |
| 腿 | 6x14 单位 | 短腿小脚 |
| 手臂 | 6x14 单位 | 自然下垂或配合动作 |

#### 6 种角色设计

| 角色 | 发型 | 服装特征 | 配饰 | 主色 |
|------|------|---------|------|------|
| 老板 boss | 偏分短发，藏青色 | 西装外套 + 领带 | 保温杯 | #4F46E5 |
| PM | 短发利落，棕色 | 衬衫 + 马甲 | 平板（手中） | #F59E0B |
| 开发 developer | 乱发/帽子，黑色 | 帽衫 + 耳机 | 机械键盘图案 | #10B981 |
| 设计师 designer | 长发/马尾，粉棕 | 贝雷帽 + 围巾 | 画笔 | #EC4899 |
| 测试 tester | 运动短发 | 运动外套 | 放大镜 | #3B82F6 |
| 员工 worker | 标准短发 | 普通衬衫 | 文件夹 | #8B5CF6 |

#### 面部表情系统（5 种）

| 表情 | 眼睛 | 嘴巴 | 状态 |
|------|------|------|------|
| 平静 neutral | 大圆眼 + 高光 | 小弧线 | idle |
| 专注 focused | 半闭 + 眉毛下压 | 一字线 | working |
| 开心 happy | 弯月形 ^_^ | 上弧微笑 | 完成任务 |
| 担忧 worried | 圆眼 + 汗滴 | 波浪线 | submitting |
| 思考 thinking | 看向一侧 | 小圆 O | 复杂任务 |

#### 动画状态（5 种，SVG 属性驱动）

| 动画 | 效果 | 参数 |
|------|------|------|
| idle | 轻微上下呼吸 | translateY: ±2px, 周期 2s |
| working | 打字动作 + 头部微点 | 手臂: ±10° 旋转, 头: ±3px |
| meeting | 手势动作 + 点头 | 手臂: ±15°, 头: ±5px |
| walking | 腿臂交替摆动 | 腿: ±20° 旋转, 臂: ±15° |
| submitting | 紧张抖动 | translateX: ±1px 高频 |

## 6. 技术约束

### 必须保持的接口

```typescript
// IsoEngine.ts — 等轴测投影（不可改）
export function toIso(x: number, z: number): { x: number; y: number }
export function sortKey(x: number, z: number, y: number): number

// IsoScene.tsx — 入口组件接口（不可改 props）
// 从 store 读取：projectRooms, approvalRequests, theme

// IsoCharacter — 必须支持这些 props:
interface Props {
  name: string
  role: TeamRole        // boss | pm | developer | designer | tester | worker
  color: string         // 角色主色
  activity: AgentActivity // idle | working | meeting | walking | submitting
  facing: 'left' | 'right'
  animT: number         // 动画时间参数 (0~1 循环)
  isWalking: boolean
  onClick?: () => void
}

// IsoFurniture — 必须支持:
type FurnitureType = 'desk' | 'monitor' | 'chair' | 'roundTable' | 'sofa' | 'plant' | 'coffeeTable'

// IsoRoom — 必须支持:
interface RoomProps {
  cx: number; cz: number; width: number; depth: number; label?: string
}
```

### SVG 性能要求

- 角色 SVG 不超过 200 个元素（当前约 50 个）
- 家具 SVG 不超过 80 个元素
- 总场景（10 角色 + 30 家具）不超过 3000 SVG 元素
- 动画仅用 `transform` 和 `opacity`（CSS 硬件加速）
- 不使用 SVG filter 做实时动画（性能杀手），仅用于静态阴影

### 主题适配

```typescript
// 当前方案：从 store 读取 theme，三元表达式选色
const theme = useAppStore((s) => s.theme)
const floorFill = theme === 'cyberpunk' ? '#1f0035' : theme === 'light' ? '#f4f4f5' : '#27272a'
```

新设计必须为每种家具/角色定义 dark / light / cyberpunk 三套色板。
