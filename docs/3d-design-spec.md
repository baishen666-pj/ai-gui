# AI GUI — 3D 虚拟办公室设计规范

> **版本:** 1.0  
> **日期:** 2026-05-13  
> **承接团队:** Hermes AI 视频项目组  
> **风格方向:** 动漫/卡通风格（Q版比例）  
> **场景设定:** 公司办公环境 · 工作日白天

---

## 1. 概述

AI GUI 是一个多 Agent 桌面工作台应用，3D 虚拟办公室以可视化方式展示 Agent 团队的实时工作状态。当前版本使用纯几何体（球体+胶囊+圆柱）构建角色，视觉效果粗糙，需要全面升级为专业动漫/卡通风格的 3D 场景。

### 核心目标

- 角色能一眼区分不同职位（老板/项目经理/开发/设计/QA 等）
- 办公环境真实可信，有公司氛围
- 动画流畅自然，支持多种工作状态
- 在 Electron 桌面应用中保持 60fps 流畅渲染

---

## 2. 美术方向

### 风格参考

- **比例:** Q版 / Chibi 风格，头身比 1:2 ~ 1:2.5（大头小身）
- **轮廓:** 清晰的描边轮廓线（outline pass），粗细 1-2px
- **材质:** Toon/Cel-shading 渲染，色块分明，不要写实光影过渡
- **色彩:** 饱和度高，对比鲜明，每类角色有专属主色
- **氛围:** 温暖明亮，有阳光感的现代办公室

### 关键视觉原则

1. **可读性优先** — 缩放到最小尺寸时也能区分角色类型
2. **色彩编码** — 不同角色用不同主色，保持一致
3. **轮廓清晰** — 所有模型带黑色描边，确保在任何背景下都清晰
4. **表情丰富** — 通过简单面部表情传达情绪状态

## 3. 角色设计

### 3.1 角色类型

| 角色 | 英文 ID | 主色 | 外观特征 | 标志性配饰 |
|------|---------|------|---------|-----------|
| 老板 | boss | #4F46E5 (靛蓝) | 西装/西装外套，领带，偏壮实 | 公文包 / 保温杯 |
| 项目经理 | pm | #F59E0B (琥珀) | 衬衫+马甲，干练形象 | 平板电脑 / 耳麦 |
| 开发工程师 | developer | #10B981 (翠绿) | 连帽衫/T恤，眼镜可选 | 机械键盘 / 双显示器 |
| 设计师 | designer | #EC4899 (粉红) | 时尚穿搭，围巾/贝雷帽 | 画板 / 调色板 |
| 测试/QA | tester | #3B82F6 (蓝色) | 运动风格，休闲装 | 放大镜 / 检查清单 |
| 普通员工 | worker | #8B5CF6 (紫色) | 标准办公着装 | 文件夹 / 咖啡杯 |

### 3.2 角色规格

- **身高:** 1.6-1.8 单位（Q版比例）
- **头身比:** 1:2（大头占全身 1/3）
- **面部:**
  - 眼睛：两个大椭圆 + 高光点（2个白色小圆）
  - 嘴巴：简单弧线（开心/平静/思考三种表情）
  - 腮红：两个半透明粉色圆（可选，增加可爱感）
- **头发:** 每类角色不同发型（老板偏分/PM短发/开发者乱发/设计师长发等）
- **身体:** 简化服装轮廓，通过颜色和配饰区分

### 3.3 多边形预算

- 单个角色: **800-1500 三角面**
- 头部（含面部、头发）: 约 400-600 tris
- 身体（含服装）: 约 300-500 tris
- 配饰: 约 100-300 tris
- 手部: 简化为拳/掌两种形态，不需要独立手指

## 6. 动画规范

### 6.1 动画状态

| 状态 | 触发条件 | 动画描述 | 时长 |
|------|---------|---------|------|
| idle | 站立不动 | 呼吸起伏（胸部），偶尔眨眼，轻微重心偏移 | 循环 |
| working | 坐在工位 | 打字（手指微动），偶尔看屏幕侧头，喝咖啡 | 循环 |
| meeting | 坐在会议室 | 点头/摇头，举手，翻看材料 | 循环 |
| walking | 移动中 | 自然行走循环（臂腿交替，身体微摆） | 速度可调 |
| standing | 站立等待 | 重心左右切换，看手表/手机 | 循环 |
| reacting | 特殊事件 | 鼓掌（庆祝）/ 拍桌（沮丧）/ 捂脸（无奈） | 1-2s 一次 |

### 6.2 面部表情

| 表情 | 眼睛 | 嘴巴 | 适用状态 |
|------|------|------|---------|
| 平静 | 正常圆眼 | 水平线 | idle, working |
| 开心 | 弯月形 ^_^ | 上弧微笑 | 完成任务 |
| 思考 | 看向一侧 | 小圆 O | working（复杂任务） |
| 惊讶 | 大圆 | 大圆 | 收到意外结果 |
| 疲惫 | 半闭 | 下弧 | working（长时间） |

### 6.3 过渡

- 状态切换使用 0.2s 混合过渡（crossfade）
- 行走速度: 1.5-2.0 单位/秒
- 行走时身体朝向移动方向，使用 easeInOutCubic 缓动

## 7. 技术规格

### 7.1 模型格式

- **生产格式:** glTF 2.0 (.glb binary)
- **源文件:** Blender (.blend)
- **动画:** 内嵌在 .glb 中，按状态命名（idle, working, meeting, walking, standing, reacting）

### 7.2 纹理规格

| 用途 | 尺寸 | 格式 | 说明 |
|------|------|------|------|
| 角色贴图 | 512 x 512 | PNG (RGBA) | Diffuse + Alpha |
| 家具贴图 | 256 x 256 | PNG | Diffuse |
| 环境（地板/墙） | 1024 x 1024 | PNG | Tiling 贴图 |
| 描边 | — | 程序化 | Shader 生成 |

### 7.3 材质

- **角色:** Toon/Cel-shading 材质（2-3 级色阶），outline pass（法线外扩法）
- **家具:** Standard 材质 + 简化纹理（木纹/织物/金属），roughness 0.5-0.8
- **地板:** Tiling 木纹贴图，roughness 0.4
- **屏幕:** Emissive 材质，亮度随主题变化

### 7.4 性能预算

| 指标 | 目标 |
|------|------|
| 帧率 | 60 FPS |
| 角色 | 最多 10 个同时显示 |
| 场景总三角面 | < 30,000 tris |
| Draw Calls | < 100 |
| 纹理内存 | < 32 MB |
| 阴影 | 1 个 DirectionalLight 阴影（1024px） |

## 9. 集成指南

### 替换步骤

1. **替换 AgentCharacter.tsx**
   - 当前使用 `<capsuleGeometry>` + `<sphereGeometry>` + `<cylinderGeometry>` 手工拼装
   - 改为使用 `useGLTF` + `useAnimations` 加载 .glb 模型
   - 保留现有 activity 调度逻辑（每 8 秒随机切换活动）

2. **替换 OfficeFurniture.tsx**
   - 当前每个家具组件手工拼 BoxGeometry/CylinderGeometry
   - 改为使用 `useGLTF` 加载对应 .glb 文件
   - DynamicFurniture 组件保持不变（从 store 读 layout items）

3. **更新 OfficeLighting.tsx**
   - 启用 `castShadow` 在 DirectionalLight
   - 添加 shadow-map 配置
   - 调整光照参数匹配新配色

4. **添加描边效果**
   - 在 Canvas 组件中添加 `<EffectComposer>` + outline pass
   - 或使用 `<mesh>` 的 `material.clippingPlanes` 方案

5. **更新 constants.ts**
   - 替换 `COLORS` 对象为新的暖色系配色
   - 更新 `FURNITURE_META` 的颜色值

### 依赖包

```bash
npm install @react-three/drei  # 已有：useGLTF, useAnimations
# 描边后处理（可选）：
npm install @react-three/postprocessing
```
